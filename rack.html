⚙ One-time database setup
Before first sync, create the rack tables. Open Supabase → SQL Editor → New query → paste this → Run. You only do this once.
create table if not exists rack_assignments (
  product_id bigint primary key,
  rack text not null,
  updated_at timestamptz default now()
);
create table if not exists rack_transactions (
  id bigint generated always as identity primary key,
  product_id bigint, product_name text,
  type text, qty integer default 0,
  rack text, app_user text,
  dest text, dest_name text,
  created_at timestamptz default now()
);
alter table rack_assignments enable row level security;
alter table rack_transactions enable row level security;
create policy "anon ra" on rack_assignments
  for all using (true) with check (true);
create policy "anon rt" on rack_transactions
  for all using (true) with check (true);
How the rack system works
1
Run the SQL above (once)Creates the rack tables in your database so locations and logs sync across all devices.
2
Print rack QR codesDownload the PDF below and stick one QR on each physical rack (Rack 1, Rack 2…).
3
Assign products in SetupScan each product's existing barcode (unchanged), tap which rack it sits on.
4
Scan rack QR anytimeFrom the Racks tab, scan a rack's QR to instantly see every product on it and its full transaction log.
5
Fast stock in / outScan a product barcode — Stock In shows which rack to put it back on, Stock Out shows which rack to pick from.
